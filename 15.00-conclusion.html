Chapter 15.

Conclusion

Over the course of this book we’ve explicitly covered a lot of topics, including routing, templating, working with a database, authentication/authorization, using HTTPS, using Go’s testing package and more.

이 책을 통해 라우팅, 템플릿, 데이터베이스 작업, 인증/인가, HTTPS 사용, Go의 테스팅 패키지 사용 등 다양한 주제를 명시적으로 다루었습니다.

But there have been some other, more tacit, lessons too. The patterns that we’ve used to implement features — and the way that our project code is organized and links together — is something that you should be able to take and apply in your future work.

하지만 명시적으로 다룬 주제 외에도 몇 가지 암시적인 교훈이 있었습니다. 기능을 구현하는 데 사용한 패턴과 프로젝트 코드의 구조, 그리고 코드 간의 연결 방식은 향후 작업에서 적용할 수 있는 것입니다.

Importantly, I also wanted the book to convey that you don’t need a framework to build web applications in Go. Go’s standard library contains almost all the tools that you need… even for a moderately complex application. For the times that you do need help with a specific task — like session management, routing or password hashing — there are lightweight and focused third-party packages that you can reach for.

중요한 것은 이 책을 통해 Go에서 웹 애플리케이션을 구축하는 데 프레임워크가 필요하지 않다는 점을 전달하고 싶었습니다. Go의 표준 라이브러리에는 거의 모든 필요한 도구가 포함되어 있습니다. 적당히 복잡한 응용 프로그램에 대해서도 그렇습니다. 특정 작업에 도움이 필요한 경우 세션 관리, 라우팅 또는 비밀번호 해싱과 같은 작업에 대한 경량이면서도 특화된 서드파티 패키지가 있습니다.

At this point, if you’ve coded along with the book, I recommend taking a bit of time to review the code that you’ve written so far. As you go through it, make sure that you’re clear in your mind about what each part of the codebase does, and how it fits in with the project as a whole.

이 시점에서 책에 따라 코드를 작성했다면, 이제 소요된 시간을 투자하여 작성한 코드를 검토하는 것이 좋습니다. 코드베이스의 각 부분이 무엇을 하는지, 그리고 전체 프로젝트와 어떻게 조화를 이루는지에 대해 명확하게 이해하도록 하세요.

You might also want to circle back to any parts of the book that you found difficult to understand the first-time around. For example, now that you’re more familiar with Go, the http.Handler interface chapter might be easier to digest. Or, now that you’ve seen how testing is handled in our application, the decisions we made in the designing a database model chapter might click into place.

또한 처음에 이해하기 어려웠던 책의 어떤 부분이라도 다시 확인해 볼 수 있습니다. 예를 들어 이제 Go에 더 익숙해졌으므로 http.Handler 인터페이스 챕터를 이해하기가 더 쉬워질 수 있습니다. 또는 애플리케이션에서 테스트가 처리되는 것을 보았으므로 데이터베이스 모델 설계 챕터에서 내린 결정이 이해하기 쉬워질 수 있습니다.

If you bought the Professional Package version of this book, then I strongly recommend working through the guided exercises in chapter 17 (right at the end of this book, after this conclusion). The exercises should help to consolidate what you’ve learned — and working through them semi-independently will give you some extra practice with the patterns and techniques from this book before you use them again in your own projects.

만약 이 책의 Professional Package 버전을 구매했다면, 권장 사항으로는 챕터 17의 안내형 연습문제를 진행하는 것을 강력히 권합니다(이 책의 마지막에 위치한 이 결론 이후에 있습니다). 연습문제를 통해 학습한 내용을 정리하는 데 도움이 되며, 이를 반 독립적으로 해결하면 본 책에서 다룬 패턴과 기술에 대한 추가적인 연습을 할 수 있어 향후 프로젝트에서 사용할 때 도움이 될 것입니다.
